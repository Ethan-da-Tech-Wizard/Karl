# Rigorous Risk Register

| Risk ID | Risk Category | Risk Description | Impact | Likelihood | Mitigation Strategy & Implementation Details |
|---------|---------------|-------------------|--------|------------|----------------------------------------------|
| **R01** | Threading | **GUI Freezing on Long Inferences:** `llama_cpp` runs a blocking C-loop. Calling `create_chat_completion` on the main PyQt thread will cause Windows to label the app "Not Responding." | Critical | High | Wrap the `create_chat_completion` call inside a `QThread` and use PyQt Signals (`pyqtSignal(str)`) to stream yielded tokens back to the main thread's `QTextBrowser`. |
| **R02** | Memory Mgmt | **VRAM/RAM Overflow:** DeepSeek 7B requires ~4.5GB RAM at Q4 quantization. Spikes during heavy context ingestion can crash the Python process entirely with an Out-of-Memory (OOM) OS kill. | Critical | Medium | Expose `n_gpu_layers` and `n_ctx` constraints in the UI. Catch `MemoryError` during initialization. Advise users to monitor system RAM and default to Q4_K_M GGUF formats. |
| **R03** | Data | **Context Window Exhaustion:** As chat history grows, it will exceed the 4096 or 8192 token limit of the model, leading to `llama_cpp` crashing or hallucinating. | High | High | Implement strict token tracking using `llama_cpp.tokenize()` on every turn. Implement a sliding window algorithm in `interaction_loop.py` that preserves the System Prompt but drops the oldest `[user, assistant]` message pairs when approaching the limit. |
| **R04** | User Error | **Catastrophic Core Modifications:** The user edits `core/interaction_loop.py` and creates a syntax error or infinite loop, breaking the application. | High | High | Wrap the execution call in `app/engine/llm_thread.py` with a global `try/except Exception as e`. Format the traceback and emit it to the chat UI as a red system error. For infinite loops, implement a "Kill Generation" button that terminates the `QThread` forcefully. |
| **R05** | Build | **`llama-cpp-python` Compilation Failure:** Installing this library from `pip` on Windows often fails if Microsoft C++ Build Tools are missing, alienating users. | Medium | High | Document the exact pre-compiled wheel installation command (`pip install llama-cpp-python --extra-index-url https://abetlen.github.io/llama-cpp-python/whl/cpu`) in the README, avoiding local C++ compilation unless the user specifically wants GPU support. |
| **R06** | Data | **FAISS Index Corruption:** Sudden app closure during RAG document ingestion leaves the `.faiss` index and the `meta.db` out of sync. | Medium | Low | Implement transaction logic: Write embeddings to FAISS, then execute SQLite commit. If either fails, rollback both. Use SQLite WAL mode to prevent locking. |
| **R07** | Privacy | **Accidental Telemetry Leak:** Underlying libraries (like `huggingface_hub` used by `sentence-transformers`) attempt to phone home to check for model updates. | High | Medium | In `main.py`, forcefully inject environment variables before importing any other libraries: `os.environ["HF_HUB_OFFLINE"] = "1"` and `os.environ["HF_HUB_DISABLE_TELEMETRY"] = "1"`. |
